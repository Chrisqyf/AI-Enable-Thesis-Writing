# 1、引言

当前，AI工具正以前所未有的速度渗透到学术写作的各个环节。从文献调研、代码实现、图表绘制到文字润色，几乎每个步骤都能找到相应的AI辅助方案。然而，工具琳琅满目，真正能把AI用好、用对的人却并不多。许多人要么将其当作"一键生成器"，结果输出内容空洞、错误频出；要么因初期效果不佳而放弃探索，错失效率提升的机会。

本人在毕业论文撰写过程中，深度使用了 Gemini、DeepSeek 等多款AI工具，覆盖了绪论撰写、方法论实现、结果分析、绘图辅助、表达润色等多个环节，踩过不少坑，也积累了一些切实可用的经验。同时，也参考了身边同学的高效实践。

这篇帖子无意贩卖"AI替代写作"的焦虑或幻觉，而是希望以**抛砖引玉**的方式，分享我们在真实写作场景中的**具体流程、关键问题及应对方案**。无论你是正在准备毕业论文，还是对AI辅助科研感兴趣，相信都能从中找到一些可借鉴的思路。

下面是正文——从绪论撰写开始，逐一拆解。

# 2、绪论撰写

从撰写逻辑来看，这部分是最后完成的。当然也可以直接基于开题时写的文献综述。

以下是我用AI辅助撰写的流程：
1. 提供全局context，比如具体的研究内容、创新点等
2. 将上述内容提供给全局对话窗口，在这里和AI一起讨论梳理需要综述的各个子话题，确保每个子话题都是明确的。可以让AI对你进行多轮提问，然后逐条回复或提出新的需求
3. 和AI进行讨论，完成各个子话题的提示词撰写
4. 针对各个子话题分别在新对话窗口执行deep research，输出调研报告。其实对应于不同子话题的文献综述/国内外研究现状，或是研究背景的内容。

对于除文献综述外的其余部分
如：摘要、标题润色、章引言/小节等部分都可以在全局chat中去完成
这些部分往往涉及全文的关联性、逻辑性，且工作量较小，往往1-2次对话就可以完成
不会过分污染context，同时增强AI对全文的理解

以下是我在实践过程中遇到的关键问题和相应的解决方案：
1. 文献幻觉 标题/作者对不上，或者完全不对

对于AI输出的文献，需要一一核对，即使提供了可供点击跳转的链接。
根据最近的实践经验，可以使用其它ai工具进行自动化的交叉验证和修改，如deepseek。把原文和参考文献列表提供给另一个AI，让它逐条检查并修正。这样会比自己检查修正要省时省力，也能保证结果的可靠性。
另外，对于想要为原文某个论点找到对应参考文献的情况，可以将原文提供给ai进行搜索，并限定检索要求。我目前用deepseek做了很多次这方面的测试和实践，效果很不错。
最后，即使AI给出的不对，也可以根据AI给出的提示词/关键词或者相关错误题目等，自己去Web of Science或者谷歌学术等进行搜索，大概率能找到目标参考文献。

2. 输出的报告内容冗长，超出预期/篇幅要求

这个问题在我的经验中，几乎很难通过在提示词阶段约束的方式进行解决（主要针对deep research功能）。我目前主要是通过在初步报告完成后，用专门的提示词和对话窗口对指定段落进行简写。

# 4、绘图

提示词撰写，交给nano banana或者GPT Image2去绘图，现在已经有很多相关的经验贴，这里不详细介绍

下面是我的一些个人理念：毕业论文不要过渡追求绘图精美，但要确保逻辑和图示清晰易理解，据我了解，那些绘图精美/篇幅长的不一定有好的评审结果。个人认为一定要辩证地看待AI绘图，AI绘图很多时候可能比手搓更耗时费劲，前提是自己已经构思好的情况下。下面分析个人认为的原因：

1. 模态过渡

AI绘图：脑子里的逻辑/图示——文字/语言——生成图
手搓：脑子里的逻辑/图示——绘图

“寻找一个精确的逻辑概念——在这里就是对“视觉类比”进行精确的口头描述——或许是徒劳的。视觉脑区本身的连接模式可能就是对这一原理的最简单的逻辑表达或定义。 ”——冯 · 诺依曼

2. AI对用户具有较高的隐形要求

其实要用AI画出那些如顶刊配图一样的精美图片以及完美达到预期的效果，会对提示词和提供的相应context要求很高，同样需要用户对AI工具有较高程度了解和经验。对于刚开始使用AI绘图的人来说，可能要花费大量时间在前期的提示词调试，或者是效果没达到预期后带来的重新返工上。

以下是参考人2（作者朋友）的一些经验：
绘制机理图或者ToC等图时可以几种AI联用，一种负责画图（可以喂一张图让它仿造类似风格画），一种负责清晰化、去水印、改字体等。因为只用一种AI来画较为复杂的图时，由于需要反复修改，很可能出现指令失准的问题。总之，一种AI已经满足不了你的需求时，及时在此基础上向另一种寻求帮助（即将第一种AI中得到的相对满意的结果直接喂给第二种AI）。

# 5、结果分析

底层逻辑：结果分析要基于方法设计和领域知识，能够清晰说明结果背后的逻辑和原理
具体做法：
1. 提供context——方法论、创新点等表述。一定要详细，给出横纵坐标以及各符号所代表的意义，通过提出具体问题的方式得到想要的分析结果。一定不要想当然觉得是废话，你以为的废话可能正是AI想得到的确切的肯定的指令。
2. 提供结果图，让AI基于context，解析结果，并从方法原理角度分析结果为什么是这样

# 6、表达风格润色

很多模型在回答时都倾向于“谄媚”或“夸大”，使用一些极端、绝对、夸张的表达，比如，彻底打破、完全、惊人、极其。这样的语风是与学术科研强调的严谨、客观、辩证的理念是完全不符的。下图是作者的论文初稿中出现的“AI痕迹”，也是被导师重点指出。

2. 在使用AI撰写内容时，前期通过“系统级提示词”约束，或者先完成内容后期再用专门的提示词进行润色。

# 7、模型选型

本人在论文写作期间主要使用的模型是 Gemini 和 DeepSeek。一个很重要的原因是“成本”。
DeepSeek在线上平台的对话完全免费，Google AI Studio中也可以用到免费的高级模型，而本人也通过一些“魔法”（淘宝、学生认证）搞到了Gemini的一年pro会员。

但在模型的具体表现上，作者有以下经验：

1. Gemini 喜欢琢磨用户的意图倾向，谄媚，观点类问题/任务表现不佳，但在执行具体任务时效果还可以。最近推出的3.5 Flash模型目前体验也还不错。此外，Gemini的“记忆”功能有时也会很影响用户体验，它会根据过往的所有对话建立你的画像，然后在后续的对话中基于这些背景去输出内容，就会导致很多时候都会往一些“片面的背景”上去扯。
2. deepseek/gpt 在“模型性格”方面更适合我，也比较符合科研风格。可以进行一些idea层面的讨论
3. 在学术文献调研方面，据说perplexity会更专业，但我目前还没怎么用过，后续准备尝试

因为我只能对有更多使用经历的模型有用后体验和评价，所以仅供参考。

# 8、未来展望

对于未来的论文撰写最大的一个期望是，可以实现真正的“vibe writing”，而不是通过对话框去复制粘贴。

microsoft office前段时间已经和anthropic达成合作，在word、ppt、excel等办公软件中接入claude code辅助办公，但大多普通用户依然很难接触到

现在也有很多的办公skills（docx、ppt-master等），但据我的使用经验，它们当前依然很难全功能且不出错地处理word文档（比如格式、公式处理）

我目前在obsidian等笔记管理平台中初步实现了 vibe writing，在侧栏对话框和AI进行问题讨论或执行检索任务，最后让AI把对话内容自动更新到笔记中，并设置相应的格式。但这些平台主要基于markdown格式进行记录，无法直接对应学术论文要求（word、latex等），或许可以通过个专门插件或skill进行格式转换
